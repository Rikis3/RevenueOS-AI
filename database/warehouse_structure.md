# RevenueOS Data Warehouse Structure (DuckDB)

This document outlines the Medallion Architecture data modeling strategy for RevenueOS. All transformations occur within DuckDB to simulate a modern cloud data warehouse (like Snowflake or BigQuery).

## Medallion Layers

### 1. Bronze Layer (Raw)
The Bronze layer consists of raw data generated by the synthetic script and exported as CSV files to `/database/bronze/`. These files are then ingested directly into DuckDB `raw_*` tables.

*   `raw_accounts`
*   `raw_contacts`
*   `raw_leads`
*   `raw_campaigns`
*   `raw_opportunities`
*   `raw_opportunity_history`
*   `raw_attribution_touchpoints`
*   `raw_sdr_activities`
*   `raw_sales_users`

### 2. Silver Layer (Conformed Star Schema)
The Silver layer cleans, deduplicates, and structures the raw data into a dimensional model (Star Schema). These are instantiated as Views in DuckDB.

**Dimensions (`dim_*`)**
*   `dim_date`: Central calendar table for time-series analysis.
*   `dim_region`: Hierarchical geographic data (Global -> Region -> Sub-Region).
*   `dim_campaign`: Marketing campaign metadata (Budget, Channel, Goal).
*   `dim_marketing_channel`: Lookup table for marketing channels (e.g., 'LinkedIn Ads', 'Organic Search').
*   `dim_sales_team`: SDR and AE organizational hierarchy and roles.
*   `dim_account`: Cleaned company demographics (Employees, Revenue, Tier, Industry).
*   `dim_contact`: Cleaned individual data linked to Accounts.

**Facts (`fct_*`)**
*   `fct_leads`: Lead lifecycle events (created, MQL'd, SQL'd).
*   `fct_opportunities`: Core pipeline table with ARR, MRR, ACV, TCV, Stage, Forecast Category.
*   `fct_opportunity_history`: Timestamped stage changes for pipeline velocity.
*   `fct_attribution_touchpoints`: The bridge between Marketing (Campaigns/Channels) and Sales (Leads/Opps).
*   `fct_sdr_activities`: Granular log of Calls, Emails, Meetings, Demos.

### 3. Gold Layer (Analytics Ready)
The Gold layer aggregates data from the Silver schema to support BI, the AI Copilot, and executive reporting. These are complex materialized views or tables in Phase 3.

*   `gold_pipeline_velocity`: Daily snapshots of pipeline movement.
*   `gold_campaign_roi`: Aggregated spend vs. attributed revenue.
*   `gold_lead_scoring`: Propensity model inputs.
*   `gold_account_scoring`: ABM intent and engagement scoring.
*   `gold_forecast`: Revenue forecasting aggregating `commit`, `upside`, and `best_case`.
*   `gold_executive_kpis`: High-level metrics (ARR, CAC, Win Rate).

## Modeling Decisions
1.  **DuckDB Engine**: DuckDB is utilized for its exceptional performance with analytical (OLAP) queries on local files. It allows us to use standard SQL without running a heavy database server.
2.  **Star Schema**: We explicitly separated `dim_sales_team`, `dim_campaign`, and `dim_region` from the fact tables to ensure dimensional analysis works flawlessly in standard BI tools.
3.  **Forecast Categories**: Opportunities use standard SaaS forecasting terminology (`Closed`, `Commit`, `Best Case`, `Pipeline`, `Omitted`) to support Phase 3 modeling.
